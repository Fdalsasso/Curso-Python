Comandos:
    cuidado al cambiar de branch:
        git switch <name of branch>
        git checkout <name of branch> (ojo con esto)

  
    git init (inicia proyecto)
    git add <files> (un punto "." agrega todo)
    git commit -m "<message>"
   
    git log --oneline (muestra todos los cambios)
    git reset --hard <codigo de commit> (vuelve a la version especificada por el codigo de commit y "borra" los commits posteriores)
    
    habia un comando que mostraba la lista de branches pero no mas
    git branch <name of new branch> (crea nueva branch)
    git branch -d <name of branch> (elimina branch)
    
    git stash save "<name>" (guarda el proceso de una branch)
    git stash list (muestra las instancias de stash)
    git stash pop <nombre de stash que aparece en list stash@{x}> (una vez que se hace pop, desaparece)
    git stash clear (limpia las instancias de stash)
    
    merge: siempre estar parado en main
        git merge <nombre de la branch> (los conflictos se pueden ver con vscode o alguna herramienta grafica)
        git log --all --decorate --oneline --graph (muestra los logs de las branches y los merges)
        
        
    git config --global alias.adog "log --all --decorate --oneline --graph" (alias global, el alias seria "adog") 
    
    git tag <name> -m "<message>" 
    git push --tags (empaqueta todo el proyecto, es como un versionado)
